# 4.1.10 Configure BIOS/UEFI Security

## Objective
Configure BIOS/UEFI security settings to protect a workstation from unauthorized access and hardware tampering.

## Tasks Performed
- Restarted the system and entered BIOS/UEFI Setup.
- Configured an administrator password to restrict BIOS configuration changes.
- Set a hard drive password to protect data access if the drive is removed from the system.
- Enabled Trusted Platform Module (TPM) security.
- Enabled chassis intrusion detection to generate alerts during POST if the system case is opened.
- Saved BIOS/UEFI configuration changes and restarted the system.
- Verified the hard drive password prompt during system startup.

## Skills Demonstrated
- BIOS/UEFI Administration
- Hardware Security Configuration
- TPM Configuration
- Drive-Level Security
- Physical Security Controls
- Security Hardening
- System Startup Verification

## Technologies Used
- BIOS/UEFI Firmware
- Trusted Platform Module (TPM)
- Hard Drive Password Protection
- Chassis Intrusion Detection

## Outcome
Successfully secured the workstation by implementing BIOS/UEFI security controls including administrator authentication, hard drive password protection, TPM activation, and chassis intrusion detection.
