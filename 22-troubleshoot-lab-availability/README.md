# CompTIA A+ Lab 4.2.10 – Troubleshoot Drive Availability

## Objective
Diagnose and resolve a system boot issue by verifying drive detection in BIOS/UEFI settings and correcting the boot sequence to ensure the operating system loads successfully.

## Skills Demonstrated
- BIOS/UEFI Configuration
- Boot Sequence Management
- Drive Availability Troubleshooting
- Hard Drive Verification
- System Startup Diagnostics
- Hardware Troubleshooting
- Operating System Boot Verification
- Desktop Support

## Tasks Completed
1. Powered on the system and accessed BIOS/UEFI settings.
2. Verified that all installed storage drives were recognized by the motherboard.
3. Inspected the configured boot sequence.
4. Identified that the internal hard drive was not included as a boot device.
5. Added the internal hard drive to the boot sequence.
6. Configured the internal hard drive as the primary boot device.
7. Saved BIOS changes and restarted the system.
8. Verified successful boot into Windows.
